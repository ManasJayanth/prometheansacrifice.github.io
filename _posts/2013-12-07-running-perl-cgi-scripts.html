Common Gateway Interface

CGI is an interface which tells the web server how to pass data back and forth to and from an application. Its allows rendering of dynamic web pages. Files requested from the special cgi-bin directory should not simply be read and sent, but instead should be executed.

Installation

CGI is actually a program called by the web server. This means a web server must be installed to run CGI scripts. In this post I assume you have installed Apache as your web server. Instructions to install Apache may be found here

Inorder to run CGI scripts, Apache must be configured to allow CGI execution. More info here

Open the configuration file default in /etc/apache2/sites-available/

[code language="javascript"]sudo gedit /etc/apache2/sites-available/default
[/code]

Search for the keyword ScriptAlias (Use Ctrl + F)
You should see an entry like this

[code language="javascript"]ScriptAlias /cgi-bin/ /usr/lib/cgi-bin/
<Directory "/usr/lib/cgi-bin">
AllowOverride None
Options +ExecCGI -MultiViews +SymLinksIfOwnerMatch
Order allow,deny
Allow from all
</Directory>
[/code]

This mean Apache has been configured to execute scripts from /usr/lib/cgi-bin/

Place your files in this directory and give execute permissions for the web server.

[code language="javascript"]sudo chmod a+x filename.pl
[/code]

Enter the url as http://localhost/cgi-bin/filename.pl as see the results.
